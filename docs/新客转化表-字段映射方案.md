# 2.新客转化表 · 字段映射方案

> 目标：生成 `2026年Youro运营数据周分析表（*.xlsx）` → **「2.新客转化表」** 月度汇总数据。  
> 与 **6.周新客订单表（Youro）** / **4.周新客订单表（Ronchamp）** 联动，且必须按 **屿路 / 镕川** 分店铺。  
> **业务规则总览**：[`业务规则汇总.md`](业务规则汇总.md)

---

## 1. 店铺拆分（前置规则）✅

### 1.1 问题

- API `company` 字段 **不可靠**（本周 6 单首单均为 `0`，但 A02 有屿路/镕川之分）
- 原脚本只写 Youro 周表，**未写 Ronchamp**

### 1.2 判定规则

| 优先级 | 来源 | 字段 | 映射 |
|--------|------|------|------|
| 1 | A02 订单信息表 | 所属店铺（F 列） | 屿路 → **Youro**；镕川 → **RonChamp** |
| 2 | API | `company` | 仅作兜底（不推荐） |

### 1.3 输出分流

| 所属店铺 | 周分析文件 | 新客订单 Sheet |
|----------|------------|----------------|
| 屿路 | `2026年Youro运营数据周分析表（*.xlsx` | **6.周新客订单表** |
| 镕川 | `2026年Ronchamp运营数据周分析表（*.xlsx` | **4.周新客订单表** |

**2.新客转化表** 仅在 **Youro 周分析表** 中，A 列区分 `Youro` / `RonChamp`，两行一组。

### 1.4 本周示例（6.22—6.28 首单）

## 3. 数据来源

### 3.1 流量侧（D/E/F）— TM或询盘

| 字段 | 来源 |
|------|------|
| D 6月新客数量 | A05 **对应店铺** 新流量表：`添加日期 ∈ [月初, 月末]`，按业务员计数（**不含 A060x**） |
| E L1+新客数量 | 同上，**非 L0**（含 L1+、L2、L3…）；交叉核对时 **L2 与 L1+ 等价** |
| F L1+占比 | E / D |

**店铺对应 A05 Sheet：**

- Youro → `屿路—26年新流量表`
- RonChamp → `镕川—26年新流量表`

> **A05 vs A060x 交叉核对**：同一业务员新流量在 A05 店铺表与 A060x 汇总表可能重复出现。  
> - 转化表 D/E **以 A05 为准**  
> - **A0604（Grace）/ A0607（Lily）** 单表混有 Youro+RonChamp：优先按 A05 客户+日期对齐店铺；兜底 Grace 用 `R-TM`→镕川、`TM`→屿路，Lily 用 `询盘`→镕川、其余 `TM`→镕川  
> - 差异输出至 `review/流量交叉核对-汇总.csv` / `流量交叉核对-明细.csv`

### 3.2 成交侧（C, G—S）— 来自首单订单

| 字段 | 来源 |
|------|------|
| 订单列表 | API `firstOrder=Y`，`orderDate ∈ [月初, 月末]` |
| 所属店铺 | A02 所属店铺 → Youro / RonChamp |
| 渠道分类 | 见 §4 |
| 成交金额 RMB | `paymentReceived × exchangeRate` 或周表 I 列 |
| 毛利润 / 毛利率 | 同 6.周新客订单表 Q/R |

| 字段 | 计算 |
|------|------|
| C 总成交数 | G + L + P（三渠道本月成交新客数之和） |
| G/I/J/K | TM 渠道成交订单汇总 |
| L/M/N/O | RFQ 渠道成交订单汇总 |
| P/Q/R/S | 其他渠道成交订单汇总 |

## 4. 渠道分类规则

对每笔 `firstOrder=Y` 订单，结合流量匹配结果：

| 渠道 | 条件 |
|------|------|
| **TM或询盘** | 匹配到新流量表，且 `类型` 为 TM / 询盘 / R-TM 等（非 RFQ） |
| **RFQ** | 类型或来源含 `RFQ`；或 A060x **RFQ数据管理表** 有该客户 |
| **其他** | 流量 **Alan分配**；**A07 意向订单** 客户首单；**镕川 Sally** 无流量（公司转介）；流量来源/类型含转介绍、微信、公海、客户介绍等 |
| **未归类** | 无流量且不符合「其他」→ 计入 **C**，不填 P/Q/R/S → `review/渠道未归类.csv` |

**注意**：

- 有 TM 流量匹配时，**不因**订单备注含「微信」改为其他  
- **Alan分配** 流量（如 alex buroni）不计入 A05 的 D，仅 A0603 侧会出现；是否补录 A05 由人工决定  
- 公司转介（Sally · Belfast）：无 A05 流量 → **其他** P 列（非 TM）

---

## 5. 业务员与店铺

同一业务员可出现在 **Youro 与 RonChamp 两行**（如 Grace、Lily）：

- 流量：按 A05 **店铺 Sheet** 分别统计
- 订单：按 A02 **所属店铺** 分别归入
- RonChamp 行可能仅有 D/E（有流量无成交），C/G 为空

**Youro 常见业务员：** Ennerson, Luck, Cindy, Grace, David, Lily（部分）  
**RonChamp 常见业务员：** Lily, Sally, Grace（部分）

脚本按 **当月有流量或成交** 的业务员动态出表，顺序：

1. Youro 组（A=Youro 首行）
2. RonChamp 组（A=RonChamp 首行）
3. 总计行

## 6. 与 6.周 / 4.周新客订单表 关系

```
                    ┌─────────────────────┐
                    │ API firstOrder=Y    │
                    │ + A02 所属店铺       │
                    └──────────┬──────────┘
                               │
              ┌────────────────┴────────────────┐
              ▼                                 ▼
     屿路 → Youro                      镕川 → RonChamp
     6.周新客订单表（按周）              4.周新客订单表（按周）
              │                                 │
              └────────────────┬────────────────┘
                               ▼
                    2.新客转化表（按月汇总）
                    + A05 新流量（按店铺）
```

- **周表**：只含当周 `begin_date ~ end_date` 订单
- **转化表**：含**当月累计** `月初（week.end_date 所在月 1 日）~ week.end_date` 首单汇总  
  例：周次 6.22—6.28 → 标题 **Youro & RonChamp 业务员新客转化汇总（6.1 - 6.28）**

## 7. 配置项（config.yaml）

```yaml
week:
  begin_date: "2026-06-22"
  end_date: "2026-06-28"
  period_label: "06.22 - 06.28"

# conversion 可选；缺省则 month_begin = week.end_date 当月 1 日，month_end = week.end_date
# conversion:
#   title: "6.1 - 6.28"
```

**默认不写回周分析 xlsx**（只出 CSV，人工粘贴）。

## 8. 输出

| 文件 | 说明 |
|------|------|
| `review/6.周新客订单表-Youro.csv` | 屿路当周订单 |
| `review/4.周新客订单表-RonChamp.csv` | 镕川当周订单 |

| Youro/RonChamp xlsx | 人工粘贴 `review/` CSV |
