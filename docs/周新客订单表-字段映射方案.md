# 6.周新客订单表 · 字段映射方案

> 目标：从 ERP API + 本地 Excel 汇总数据，输出 `2026年Youro运营数据周分析表（*.xlsx）` → **「6.周新客订单表」** 所需行数据。  
> 示例周期：`2026-06-22` ~ `2026-06-28`（周标签 `06.22 - 06.28`）

---

## 1. 总体流程

```
┌─────────────────────────────────────────────────────────────────┐
│ Step 1  sales-order/list                                        │
│         firstOrder=Y + 日期范围                                  │
│         → 上周全部新客销售订单（主列表）                           │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 逐单 orderNo
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 2  purchaser-order/list                                    │
│         orderNo=<销售订单号>                                       │
│         → 采购金额、运费、成交产品明细                             │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 交叉匹配
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 3  本地 Excel                                               │
│         A02 / A05 / A-060x 等                                     │
│         → 流量、收款渠道、物流、财务校验                           │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 映射 + 计算
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ Output  「6.周新客订单表」22 列 × N 行（N = 当周新客订单数）        │
└─────────────────────────────────────────────────────────────────┘
```

**原则**

| 层级 | 说明 |
|------|------|
| 主数据源 | ERP 销售订单 API（订单主体） |
| 成本数据源 | ERP 采购订单 API（采购/运费） |
| 补充数据源 | 本地 Excel（流量运营维度、财务校验） |
| 输出模板 | 周分析表「6.周新客订单表」列结构，**不是**输入源 |

---

## 2. 输入源说明

### 2.1 API · 销售订单

- **脚本**：`curl-orders.sh`
- **接口**：`POST /comp/sales-order/list`
- **关键参数**：

| 参数 | 示例 | 说明 |
|------|------|------|
| `params[beginOrderDate]` | `2026-06-22` | 周起始 |
| `params[endOrderDate]` | `2026-06-28` | 周结束 |
| `firstOrder` | `Y` | 仅新客首单 |
| `pageSize` | 建议 `100` | 避免分页遗漏 |

- **关键响应字段**：

| 字段 | 含义 |
|------|------|
| `orderNo` | 订单号（关联键） |
| `orderDate` | 订单日期 |
| `createBy` | 销售人员 |
| `customerName` | 客户姓名 |
| `country` | 国家 |
| `qty` | 型号数量 |
| `orderAmount` | 订单总金额（USD） |
| `paymentReceived` | 实际到账（USD） |
| `productAmount` | 产品总金额（USD） |
| `exchangeRate` | 汇率（通常 6.7） |
| `payMethod` | 收款方式代码 |
| `productName` | 成交产品明细 |
| `orderExpressFee` | 订单运费（USD） |
| `actualExpressFee` | 实际运费 |
| `salesRemark` | 销售备注（常含物流方式 DHL/DDP 等） |
| `firstOrder` | 是否新客（`Y`） |
| `company` | 店铺（`0` = 屿路/Youro） |

### 2.2 API · 采购订单

- **脚本**：`curl-purchaser.sh`
- **接口**：`POST /comp/purchaser-order/list`
- **关键参数**：`orderNo=<销售订单号>`

- **关键响应字段**：

| 字段 | 含义 |
|------|------|
| `purchaseAmount` | 采购金额（RMB） |
| `productName` | 采购侧产品明细 |
| `orderExpressFee` | 订单运费 |
| `actualExpressFee` | 实际运费 |
| `internalExpressFee` | 国内运费 |
| `externalExpressFee` | 国际运费 |
| `otherFee` | 其它费用 |
| `salesRemark` | 物流备注 |
| `exchangeRate` | 汇率 |

### 2.3 本地 Excel

| 文件 | Sheet | 用途 |
|------|-------|------|
| `A02-财务收支.xlsx` | `屿路 - 订单信息表` | 财务字段校验、收款渠道、综合成本分项 |
| `A02-财务收支.xlsx` | `屿路 - 支出表` | 采购支出明细（按 ERP 订单号） |
| `A05-销售管理表.xlsx` | `屿路—26年新流量表` | 流量添加日期、来源、咨询品类 |
| `A-0601~A0607-*数据汇总表.xlsx` | `新流量表` / 各业务员表 | 与 A05 互补，按业务员维度查流量 |
| `A-0601~A0607-*数据汇总表.xlsx` | `订单跟踪管理` | 物流状态、承运商、运单号 |
| `2026年Youro运营数据周分析表（*.xlsx` | `6.周新客订单表` | **仅作输出列结构参考** |

## 3. 关联键与匹配规则

### 3.1 主键

```
orderNo（API）
  = ERP订单号（A-060x 订单跟踪）
  = 备注/关联列（A02 支出表 J 列等）
```

示例：`EN260623898`

### 3.2 流量表匹配（无订单号时）

```
normalize(createBy) + normalize(customerName)
```

| 规则 | 说明 |
|------|------|
| 大小写 | 忽略（`ennerson` → `Ennerson`） |
| 空格 | 去首尾空格 |
| 多表优先级 | `A05 屿路—26年新流量表` > 对应业务员 `A-060x 新流量表` |
| 多条命中 | 取「添加日期」最近且 ≤ 订单日期的一条 |

### 3.3 A02 财务订单匹配

```
订单日期 + 销售人员 + 客户姓名（模糊一致）
或
ERP 订单号（支出表 / 备注字段）
```

### 3.4 范围过滤

- 仅 `firstOrder = Y`
- 仅 Youro（`company = 0`）
- 订单日期落在目标周 `[begin, end]`

## 6. 转换与计算规则

### 6.1 收款渠道映射（`payMethod`）

| API 值 | 输出文本 | 备注 |
|--------|----------|------|
| `2` | 信保订单 | 与 A02 常见值一致 |
| 其他 | 查 A02 或留空待人工 | 需运行后补全枚举表 |

**规则**：A02 有值时以 A02 为准；A02 为空时用 API 映射。

### 6.2 采购金额（API 与 A02 订单信息表对比）✅ 已确认

**取值逻辑（两路并行采集，再对比）**

| 来源 | 字段 | 说明 |
|------|------|------|
| API 采购单 | `purchaseAmount` | 主写入值（写入周表 L 列） |
| A02 订单信息表 | `采购金额（人民币）` | 对比项 |

> **支出表不参与对比**（单笔支出明细与订单表采购总额口径不同，易误导）。

**对比规则**

```
api_amount = purchaser-order.purchaseAmount
a02_order  = A02 屿路-订单信息表.采购金额

写入 L 列 = api_amount（API 有值时）
         或 a02_order（API 无值时）
```

**差异提示（必做，不写进周表 22 列）**

在 **`review/采购核对.csv`** 中，每单输出：

| 核对字段 | 示例 |
|----------|------|
| `purchase_api` | 1550 |
| `purchase_a02_order` | 1550 |
| `purchase_diff` | 是 / 否 / 部分缺失 / 缺失 |
| `purchase_alert` | `API(1550) ≠ A02订单(1400)` |

**判定差异**

- 两路数值均非空，且 `|API − A02| > 0.01` → `purchase_diff = 是`
- 仅一路有值 → `purchase_diff = 部分缺失`
- 两路皆空 → `purchase_diff = 缺失`

> 周表 L 列仍写入 API 值；人工根据核对清单决定是否改 A02 或 ERP。

### 6.3 运费相关

按周表历史行推断，运费字段来源优先级：

1. A02 订单信息表（`运费`、`其它运输运费`、`初始运费`）
2. 采购 API（`internalExpressFee` / `externalExpressFee` / `orderExpressFee`）
3. 销售 API `orderExpressFee × exchangeRate`

USD 运费转 RMB 统一乘 `exchangeRate`。

### 6.4 毛利润与毛利率

根据周分析表历史数据反推：

**默认公式（多数行适用）**

```
Q（毛利润） = J（产品总金额 RMB） − L（采购金额 RMB）
R（毛利率） = Q / J
```

**扩展公式（当 M/N/O/P 有值时）**

```
Q = J − L − M − N − O − P
```

**规则**

- A02 已填 `产品毛利润` / `毛利率` 且与公式误差 < 1%：以 A02 为准
- 否则按上述公式计算
- 输出毛利率保留 3~4 位小数（如 `0.183`）

### 6.5 流量来源 ✅ 已确认

周表 `T` 列历史值为：`TM`、`转介绍`、`RFQ` 等。

| Excel 列 | 映射到 T 列 |
|----------|-------------|
| A05 `类型`（TM/询盘） | **首选** ✅ |
| A05 `来源`（国际站） | 当类型为空时使用 |

> 「国际站」是平台来源，「TM」是沟通类型；周表 `流量来源` 使用 **类型** 字段。

### 6.6 成交产品 / 品牌识别 ✅ 策略已定

**周表输出格式**（V 列，22 列之一）：

```
成交产品：{品牌英文名}
```

**设计原则**：自动给出「最合理猜测」，低置信度进复核清单，**你后期人工改 V 列即可**；脚本不阻塞、不猜不出来就留 `待确认`。

---

#### 6.6.1 四层推断（按优先级）

```
Layer 1  配置文件 brands.yaml（型号前缀 / 关键词 → 品牌）
    ↓ 未命中
Layer 2  A05 咨询品类 → 品牌（品类词典）
    ↓ 未命中
Layer 3  API productName 关键词扫描（CHINT、Siemens、SEW…）
    ↓ 未命中
Layer 4  兜底 → 待确认（V 列写「成交产品：待确认」）
```

---

#### 6.6.2 Layer 1 · `brands.yaml`（可迭代，初始内置常见规则）

```yaml
# 型号前缀（productName 中匹配）
prefixes:
  "6SL": Siemens
  "6ES": Siemens
  "6AV": Siemens
  "NXBLE": CHINT
  "AF26": ABB
  "VFD": Delta
  "SM6000": "待确认"   # 示例：未知可标待确认

# 关键词（大小写不敏感）
keywords:
  CHINT: CHINT
  西门子: Siemens
  SIEMENS: Siemens
  SEW: SEW
  Bauer: Bauer
  Mitsubishi: Mitsubishi
  三菱: Mitsubishi
  Autonics: Autonics
  奥托尼克斯: Autonics
  JW2: "待确认"

# 周表历史品牌（从既有「6.周新客订单表」V 列反向提取，启动时 seed）
# 例：成交产品：Mitsubishi → Mitsubishi
```

**维护方式**：你人工修正 V 列后，可选把 `(型号片段 → 品牌)` 追加进 `brands.yaml`，下次同型号自动命中。

---

#### 6.6.3 Layer 2 · 品类词典（A05 `品类` → 品牌）

| 品类关键词 | 输出品牌 |
|------------|----------|
| 西门子 | Siemens |
| 三菱 | Mitsubishi |
| 正泰 / CHINT | CHINT |
| SEW | SEW |
| 奥托尼克斯 / Autonics | Autonics |
| ABB | ABB |
| 施耐德 | Schneider |

品类含多个品牌时 → 置信度 `低`，进复核清单。

---

#### 6.6.4 Layer 3 · productName 扫描

- 对 API 销售单 / 采购单 `productName` 全文做关键词匹配（Layer 1 keywords）
- 对 `|` 分隔的型号段，逐段做 prefix 匹配（Layer 1 prefixes）
- **多品牌命中**（如一行里 CHINT + Siemens）→ 取出现次数最多；仍并列则 `待确认` + 复核

---

#### 6.6.5 置信度与复核清单（不写周表，单独输出）

每单在 `review/品牌复核.csv` 增加：

| 字段 | 说明 |
|------|------|
| `orderNo` | 订单号 |
| `brand_suggested` | 脚本建议品牌 |
| `brand_confidence` | `高` / `中` / `低` / `待确认` |
| `brand_source` | `prefix` / `keyword` / `category` / `fallback` |
| `productName_raw` | API 原始 productName（便于人工判断） |
| `category_raw` | A05 品类 |
| `brand_final` | **留空，供你填** |

**置信度规则**

| 条件 | 置信度 |
|------|--------|
| prefix 唯一命中 + 与品类一致 | 高 |
| 仅 keyword 或仅品类命中 | 中 |
| 多品牌冲突 / 仅模糊关键词 | 低 |
| 全无命中 | 待确认 |

**人工流程**

1. 跑脚本 → 周表 V 列先填建议值（含 `待确认`）
2. 打开 `review/品牌复核.csv`，只改 `brand_final` 或直接在 xlsx 改 V 列
3. （可选）把确认过的规则写回 `brands.yaml`

> 不要求一次建全品牌库；**先跑通、后积累** 即可。
