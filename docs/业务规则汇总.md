# Youro 周表生成 · 业务规则汇总

> **用途**：汇总脚本 `generate_weekly_new_orders.py` 与人工周分析表对齐过程中**已确认**的业务规则，供后续每周生成与维护参考。

> **详细字段映射**见：  
> - [`周新客订单表-字段映射方案.md`](周新客订单表-字段映射方案.md)

## 1. 运行与输出

| 项 | 规则 |
|----|------|
| 默认行为 | **只生成 CSV**（`review/`），**不写回**周分析 xlsx，由人工粘贴 |
| 写回 xlsx | 显式加 `--write-xlsx`（Youro → Sheet 6，RonChamp → Sheet 4） |
| 跳过转化表 | `--no-conversion` |
| 配置 | 复制 `config.example.yaml` → `config.yaml`，填入 `JSESSIONID` |

| 文件 | 说明 |
|------|------|
| `review/6.周新客订单表-Youro.csv` | 屿路当周首单（22 列） |
| `review/4.周新客订单表-RonChamp.csv` | 镕川当周首单（22 列） |

| `review/采购核对.csv` | API vs A02 采购 |
| `review/品牌复核.csv` | 品牌推断复核 |

## 2. 店铺拆分（全局前置）

| 优先级 | 来源 | 规则 |
|--------|------|------|
| **1（权威）** | A02 `屿路 - 订单信息表` · **F 列「所属店铺」** | 屿路 → **Youro**；镕川 → **RonChamp** |
| 2（兜底） | API `company` | 不推荐单独使用 |

**API `company` 不可靠**：首单可能全部为 `0`，必须以 A02 为准。

| 所属店铺 | 周表输出 | 周分析 xlsx Sheet |
|----------|----------|-------------------|
| 屿路 | `6.周新客订单表-Youro.csv` | Youro 文件 · **6.周新客订单表** |
| 镕川 | `4.周新客订单表-RonChamp.csv` | Ronchamp 文件 · **4.周新客订单表** |

同一业务员（Grace、Lily）可同时出现在 Youro / RonChamp 两组。

## 3. 新客转化表 · 日期与结构

| 项 | 规则 |
|----|------|
| 统计范围 | **当月累计**：`week.end_date` 所在月 **1 日** ~ **`week.end_date`** |
| 标题示例 | 周次 6.22—6.28 → `Youro & RonChamp 业务员新客转化汇总（6.1 - 6.28）` |
| 存放位置 | 仅在 **Youro 周分析表**；A 列区分 `Youro` / `RonChamp` |
| C 总成交 | 当月全部首单数（含未归类） |
| D/E 流量 | **仅 A05** 对应店铺新流量表，**不叠加** A060x 计数 |

## 4. 流量数据：A05 与 A060x

### 4.1 权威与用途

| 数据源 | 用途 |
|--------|------|
| **A05** `屿路—26年新流量表` / `镕川—26年新流量表` | 转化表 **D/E**、交叉核对基准 |
| **A060x** 业务员 `新流量表` | 交叉核对、周表流量匹配（辅） |

**禁止**：将 A05 与 A060x 行数简单相加（会重复计数，如 Lily 曾出现 123=57+66）。

### 4.2 A0604（Grace）/ A0607（Lily）双店铺表

两表**单 Sheet 内混有 Youro + RonChamp**，须按行区分店铺：

| 步骤 | 规则 |
|------|------|
| 1 | 用 **A05** 按「业务员 + 客户 + 添加日期」对齐店铺 |
| 2 · Grace | 兜底：`R-TM` → 镕川；`TM` → 屿路 |
| 2 · Lily | 兜底：`询盘` → 镕川；`TM` 且 A05 未命中 → 镕川；A05 命中 Youro 的 TM → 屿路 |

### 4.3 流量匹配（周表 T 列等）

- 匹配键：客户姓名 + 业务员（模糊一致）+ 添加日期 ≤ 订单日期  
- 优先 **A05**，其次 A060x  
- 匹配时须 **同店铺**（Youro 订单不对镕川流量）  
- **无流量匹配时**：T 列留空，**不默认填「转介绍」**

### 4.4 阿里分级 · L1+

| 场景 | 规则 |
|------|------|
| E 列 L1+ 计数 | **非 L0**（含 L1+、L2、L3…） |
| A05 vs A060x 交叉核对 | **L2 与 L1+ 视为等价**（如 Malek Alaydi） |

### 4.5 客户名归一（交叉核对）

| 别名 | 统一为 |
|------|--------|
| airo Mafra | Jairo Mafra |

后续可在脚本 `TRAFFIC_CUSTOMER_ALIASES` 追加。

## 5. 流量交叉核对

### 5.1 目的

A05 与 A060x 可能重复维护同一客户；核对**差异**，不用于转化表 D/E 加总。

### 5.2 输出说明

- **汇总**：展示完整 **表名 › Sheet 名** 及条数，不用「A05=35 vs A060x=83」抽象表述  
- **明细**：`a05_workbook` / `a05_sheet` / `a060x_workbook` / `a060x_sheet`  
- **状态**：一致 / 仅A05 / 仅A060x / 日期不一致 / 字段不一致  

### 5.3 常见差异处理

| 类型 | 示例 | 建议 |
|------|------|------|
| 仅 A060x | Cindy · alex buroni · Alan分配 | A05 未同步；Alan分配属「其他」流量，**不进 D**；确认是否补录 A05 |
| 拼写差异 | airo / Jairo Mafra | 统一姓名或加入别名表 |
| 分级不一致 | L1+ vs L2 | 按 §4.4 视为一致；人工择一维护 |

## 6. 成交渠道分类（转化表 G/L/P）

对每笔当月 `firstOrder=Y` 订单：

| 渠道 | 条件 |
|------|------|
| **TM或询盘** | 匹配到新流量，且类型为 TM / 询盘 / R-TM（非 RFQ、非 Alan分配） |
| **RFQ** | 类型或来源含 RFQ |
| **其他** | 见下表 |
| **未归类** | 无流量且不满足「其他」→ **计入 C**，**不填 P/Q/R/S** → 输出 `渠道未归类.csv` |

### 6.1 「其他」成交（P 列）

| 条件 | 说明 |
|------|------|
| 流量类型 **Alan分配** | 属其他渠道（流量侧；若该客户有首单且仅此流量则归 P） |
| **A07 意向订单** 有该客户 | 如 Cindy · Anisa |
| **镕川 Sally** 且无流量匹配 | 公司转介首单，如 Belfast |
| 流量来源/类型含 | 转介绍、微信、公海、客户介绍等关键词 |

**注意**：

- 订单备注含「微信」**不**覆盖已有 TM 流量匹配（如 Valeriy 收款备注）  
- 无流量时，备注关键词才参与「其他」判定  

### 6.2 未归类（典型）

- 全库无当月有效流量匹配、且不符合 §6.1「其他」规则的首单  
- 确认渠道后：**不改源 Excel**，在 `exceptions.yaml` 登记单次例外（见 §11）

## 11. 单次例外（exceptions.yaml）

**原则**：不改 A02 / A05 / A060x 等源数据；通用规则走脚本，**非共性个案**单独登记，**按 `order_no` 仅对 listed 订单生效**。

| 动作 | 配置键 | 效果 |
|------|--------|------|
| **转化表排除** | `conversion_excludes` | 不进 C/G/L/P；**周新客订单表 CSV 仍输出** |
| **渠道覆盖** | `conversion_channel_overrides` | 覆盖自动分类为 `tm` / `rfq` / `other` |

运行后输出 `review/转化表例外应用.csv` 供核对。

|----------|------|------|------|

| LU260605659 | 覆盖 | other | Luck · 离职业务员转介 |
| LU260617825 | 覆盖 | other | Luck · 离职业务员转介 |

```yaml
  exceptions: exceptions.yaml
```

## 7. 周新客订单表 · 核心计算（摘要）

| 主题 | 规则 |
|------|------|
| 订单范围 | API `firstOrder=Y` + 当周日期 |
| 流量来源 T 列 | A05 **类型**（TM/询盘），非「来源」列 |
| 毛利润 Q | 默认 `J − L`（产品 RMB − 采购）；A02 有运费分项时 `J − L − M − N − P` |
| 采购 L | 优先 API；与 **A02 订单信息表** 核对（不含支出表） |
| 品牌 | `brands.yaml` + 四层推断；低置信度见 `品牌复核.csv` |
| 公司转介 | Sally 口径：不算新流量；无表记录 → 渠道未归类或「其他」（见 §6） |

## 8. 配置项（config.yaml）

```yaml
week:
  begin_date: "2026-06-22"
  end_date: "2026-06-28"
  period_label: "06.22 - 06.28"

# 可选；缺省 month_begin = week.end_date 当月 1 日
# conversion:
#   title: "6.1 - 6.28"

paths:
  a02: A02-财务收支.xlsx
  a05: A05-销售管理表.xlsx
  a07: A07-意向和高潜订单表.xlsx
  output_dir: review
  # weekly_youro_xlsx / weekly_ronchamp_xlsx  # 仅 --write-xlsx 时需要
```

## 9. 每周操作建议

1. 更新 `config.yaml` 日期与 Cookie  
2. 运行 `python generate_weekly_new_orders.py`  
3. 检查 `流量交叉核对-*.csv`，处理仅 A05 / 仅 A060x  
4. 检查 `渠道未归类.csv`；若有个案，登记 `exceptions.yaml`（不改源 Excel）  
5. 检查 `转化表例外应用.csv` / `采购核对.csv` / `品牌复核.csv`  
6. 将 CSV **人工粘贴**到 Youro / Ronchamp 周分析 xlsx
